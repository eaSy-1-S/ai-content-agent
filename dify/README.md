# Dify 工作流说明

本目录用于保存可导入 Dify 的工作流 DSL，方便在新环境中复现公众号读书推文生成链路。

## 文件说明

| 文件 | 用途 |
| --- | --- |
| `book_article_agent.dsl.yml` | Dify Workflow 导出文件，用于根据书籍信息生成公众号读书推文。 |

## 导入方式

1. 登录 Dify 控制台。
2. 进入工作室，选择从 DSL 文件导入应用或工作流。
3. 上传 `dify/book_article_agent.dsl.yml`。
4. 根据 Dify 页面提示完成导入。
5. 检查模型供应商配置。本 DSL 使用 DeepSeek 模型插件，导入后需要在你自己的 Dify 工作区配置对应模型供应商和 API 凭据。
6. 发布工作流，并在 API 访问中获取新的 Workflow API Key。
7. 将新 API Key 填入本地 `.env` 的 `DIFY_API_KEY`，不要写入仓库。

## 输入变量

工作流入口变量如下：

| 变量名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `book_info` | paragraph | 是 | 由 `run_feishu_dify.py` 拼接的书籍与选题信息，包含书名、作者、分类、目标读者、推荐角度、关键词、标题方向和适合人群。 |

本项目会把飞书字段组装成类似下面的文本后传给 Dify：

```text
书名：...
作者：...
分类：...
目标读者：...
推荐角度：...
核心关键词：...
标题方向：...
适合人群：...
```

## 输出字段

工作流结束节点输出以下字段：

| 字段名 | 说明 | 项目使用方式 |
| --- | --- | --- |
| `article` | 初稿内容。 | 主要用于 Dify 内部流程沉淀，当前 Python 脚本不直接回写该字段。 |
| `review` | 审核或修改建议。 | 可被解析为 `review_result` 并回写到飞书字段 `审核结果`。 |
| `final_article` | 最终公众号推文。 | 项目优先读取该字段，并拆分为最终标题、文章摘要、封面文案、公众号正文和发布前检查。 |

`run_feishu_dify.py` 兼容读取 `final_article`、`final_acticle`、`final`、`article` 作为最终推文来源；推荐在 Dify 中保持 `final_article` 字段名，避免后续维护成本。

## 最终推文格式

为了让 Python 脚本稳定拆分字段，`final_article` 建议包含以下标记：

```text
【最终标题】
...
【文章摘要】
...
【封面文案】
...
【公众号正文】
...
【发布前检查】
...
```

## 安全注意事项

- DSL 文件不应包含 Dify API Key、飞书密钥、微信公众号密钥或任何真实 token。
- 导入后在 Dify 控制台配置模型供应商凭据，不要把凭据写进本仓库。
- 如果修改工作流后重新导出 DSL，请先人工检查文件中是否包含真实密钥、内部 URL 或敏感示例数据。
